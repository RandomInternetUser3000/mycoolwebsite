Accessible, reusable foundations

Component APIs are built to be accessible out of the box‚Äîfocus states, reduced motion variants, semantic markup, and WCAG-friendly color tokens.

  • Token-driven theming for color, spacing, radius, and typography.
  • Composable primitives (buttons, inputs, cards) with sensible keyboard interactions.
  • Documentation pages with live previews and usage guidelines.

Built for fast iteration

Storybook-style previews and a Vercel demo keep feedback loops short while CI checks accessibility and lint rules.

  1. Automated contrast checks on the design token palette.
  2. ARIA test cases for interactive components.
  3. Preview deploys on each pull request for design/dev review.

System contents

  • Foundational tokens: color, typography, spacing, radius, motion.
  • UI primitives: buttons, inputs, cards, alerts, nav, toasts.
  • Content patterns: hero, pricing, feature grids, FAQ, CTAs.

Planned additions

  • Design token pipeline (Style Dictionary) with multi-theme outputs.
  • Playwright a11y smoke tests for key flows.
  • Figma kit to mirror production components.
